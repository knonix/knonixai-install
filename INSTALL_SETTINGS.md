# Install settings reference

Every important environment variable for a **customer install** (public image +
[knonixai-install](https://github.com/knonix/knonixai-install)).

> **Non-technical path:** run `./install.sh` and answer the prompts — see
> [EASY_SETUP.md](./EASY_SETUP.md) in the installer package.

Values live in `.env` next to `docker-compose.yml`. Lines starting with `#` are
comments. After changing settings: `docker compose up -d` (or re-run `./install.sh`).

---

## Required vs optional

| Priority | Settings |
|----------|----------|
| **Required** | `POSTGRES_PASSWORD` |
| **Strongly recommended (connected mode)** | `KNONIX_LICENSE_SERVICE_TOKEN` |
| **Recommended for production HTTPS** | `KNONIX_DOMAIN`, `KNONIX_ACME_EMAIL` |
| **Auto-generated by install.sh** | JWT secrets, heartbeat secret, connector encryption key, Supabase/GoTrue keys |
| **Optional** | SSO, SMTP, OAuth connectors, frontier keys, Git tokens |

## Image & networking

| Variable | Default | Description |
|----------|---------|-------------|
| `KNONIX_IMAGE_TAG` | `latest` | Container tag (`v1.x.x` recommended in production) |
| `KNONIX_DOMAIN` | _(empty)_ | Public hostname → enables Caddy HTTPS |
| `KNONIX_ACME_EMAIL` | _(empty)_ | Let’s Encrypt expiry notices |
| `KNONIX_PUBLIC_URL` | auto | Full public URL (set by installer when domain is set) |
| `NEXT_PUBLIC_BASE_URL` | auto | Browser base URL for OAuth callbacks |

**No domain:** app at `http://localhost:3000`  
**With domain:** `https://<KNONIX_DOMAIN>` (ports 80/443 open; DNS A/AAAA to host)

## Local AI (Ollama) — sovereign default

| Variable | Default | Description |
|----------|---------|-------------|
| `OLLAMA_BASE_URL` | `http://ollama:11434` | Ollama service URL inside Docker |
| `OLLAMA_NUM_CTX` | `1536` (low) / `2048` (medium) / `4096` (high) | Context window; install auto-sets via hardware profile |
| `OLLAMA_NUM_PREDICT` | `256` / `512` / `1024` | Max new tokens; keep short on CPU / Mac VMs |
| `OLLAMA_MAX_LOADED_MODELS` | `1` | Avoid multi-model thrash on 16 GB hosts |
| `OLLAMA_NUM_PARALLEL` | `1` | Concurrent Ollama slots |
| `KNONIX_MODEL` | `qwen2.5:3b` (low) | Instruct model — not qwen3 thinking by default |
| `KNONIX_LICENSE_MODE` | `connected` | Use `offline` for air-gap / GCC (disables heartbeat-cron) |
| `KNONIX_AUTH_DISABLE_SIGNUP` | `false` | Set `true` after first owner on public domains |
| `OLLAMA_KEEP_ALIVE` | `-1` | Keep model loaded (`-1` = always) |
| `KNONIX_MODEL` | `qwen2.5:7b` | Default chat model |
| `KNONIX_CODING_MODEL` | `qwen2.5-coder:7b` | Default coding model |

Installer pulls chat + embed models. Optional: pull larger models via Admin or
`docker compose exec ollama ollama pull <name>`.

## Authentication

| Variable | Default | Description |
|----------|---------|-------------|
| `ENABLE_AUTH` | `true` | Multi-user accounts |
| `KNONIX_AUTH_MAILER_AUTOCONFIRM` | `true` | Skip email confirm until SMTP is set |
| `KNONIX_AUTH_JWT_SECRET` | auto | JWT signing secret |
| `NEXT_PUBLIC_SUPABASE_URL` | auto | Auth API public URL |
| `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` | auto | Browser key |
| `SUPABASE_SECRET_KEY` | auto | Server key |
| `KNONIX_AUTH_SITE_URL` | auto | Site URL for redirects |
| `KNONIX_AUTH_API_EXTERNAL_URL` | auto | External auth API URL |
| `KNONIX_AUTH_ADDITIONAL_REDIRECT_URLS` | auto | Extra OAuth redirects |

### Optional SSO

| Variable | Description |
|----------|-------------|
| `KNONIX_AUTH_GOOGLE_ENABLED` | `true` to enable Google sign-in |
| `KNONIX_AUTH_GOOGLE_CLIENT_ID` / `…_SECRET` | Google OAuth app |
| `KNONIX_AUTH_AZURE_ENABLED` | `true` for Entra |
| `KNONIX_AUTH_AZURE_CLIENT_ID` / `…_SECRET` | App registration |
| `KNONIX_AUTH_AZURE_URL` | Authority base only (no `/v2.0`) — commercial `login.microsoftonline.com/...` or GCC High `login.microsoftonline.us/...` |

Redirect URI (both): `<NEXT_PUBLIC_SUPABASE_URL>/auth/v1/callback`

### Optional SMTP

| Variable | Description |
|----------|-------------|
| `KNONIX_AUTH_SMTP_HOST` / `PORT` / `USER` / `PASS` | Mail server |
| `KNONIX_AUTH_SMTP_ADMIN_EMAIL` | From address |
| `KNONIX_AUTH_SMTP_SENDER_NAME` | From display name |

After SMTP works, set `KNONIX_AUTH_MAILER_AUTOCONFIRM=false` to require confirmation.

## Licensing & fleet

| Variable | Default | Description |
|----------|---------|-------------|
| `KNONIX_LICENSE_MODE` | `connected` | `free` · `connected` · `offline` |
| `KNONIX_FREE_SEATS` | `1` | Free seats per customer install |
| `KNONIX_PLATFORM_OWNER` | `false` | **Only** Knonix platform host may set `true` (unlimited seats) |
| `KNONIX_LICENSE_SERVICE_URL` | `https://ai.knonix.com` | Fleet board URL |
| `KNONIX_LICENSE_SERVICE_TOKEN` | _(from Knonix)_ | Fleet enrollment token |
| `KNONIX_LICENSE_KEY` | auto | Assigned after first admin sign-up |
| `KNONIX_HEARTBEAT_SECRET` | auto | Authenticates daily seat heartbeat |

### Offline / air-gap

```bash
KNONIX_LICENSE_MODE=offline
KNONIX_LICENSE_TOKEN=...        # from sales@knonix.com
KNONIX_LICENSE_PUBLIC_KEY=...
```

Heartbeat payload: opaque install hash, license key, **integer seat count**, version — **no emails, names, or documents**.

## Connectors (M365 / Google)

| Variable | Description |
|----------|-------------|
| `KNONIX_CONNECTOR_ENCRYPTION_KEY` | Auto — encrypts OAuth tokens at rest |
| `KNONIX_MS_OAUTH_CLIENT_ID` | Entra app (client) ID |
| `KNONIX_MS_OAUTH_CLIENT_SECRET` | Entra client secret |
| `KNONIX_MS_CLOUD_ENVIRONMENT` | `commercial` · `gcc` · `gcc_high` |
| `KNONIX_MS_TENANT_ID` | Optional tenant lock |

Connector redirect:

```text
https://<KNONIX_DOMAIN>/api/knonix/connectors/microsoft/callback
```

Org admins still enable providers in **Admin → Connectors**; each user links their account.

## Platform & product flags

| Variable | Default | Description |
|----------|---------|-------------|
| `KNONIX_PLATFORM_MODE` | `sovereign` | Platform posture |
| `KNONIX_STUDIO_ENABLED` | varies | Studio playbooks UI/API |
| `KNONIX_CLOUD_DEPLOYMENT` | `false` | **Self-hosted must stay `false`.** When `true`, chat **hides the model bubble/selector** and uses hosted multi-tenant behavior. Platform fleet host uses `KNONIX_PLATFORM_OWNER=true` instead — do not set cloud=true just for fleet. |

## Object storage (library file uploads)

If configured, chat/library binary uploads use S3-compatible storage:

| Variable | Description |
|----------|-------------|
| `R2_ACCESS_KEY_ID` / `R2_SECRET_ACCESS_KEY` | Credentials |
| `R2_ACCOUNT_ID` or `S3_ENDPOINT` | Endpoint |
| `R2_BUCKET_NAME` | Bucket |

Space **Vault** paste (text) works without object storage.

## After install — first-day product checklist

1. Open app → **`/auth/sign-up`** (first user = owner)
2. **`/admin`** — license, models, health
3. **`/admin/members`** — invite seats
4. **Admin → Connectors** (optional) — M365 / Google
5. **`/spaces`** — create a Space
6. **Space → Settings → SKILL.md** — personality
7. **Space → Settings → Productivity** — vault, memory, skill packs, approvals
8. Start a **space chat** and ask a real work question

API: `GET /api/knonix/setup-checklist` (signed-in) returns the same progress programmatically.
